  <action>
    Add two new migrations to the `get_migrations()` vector in `src-tauri/src/db/migrations.rs`:

    **Migration version 5** -- `create_sub_mods_table`:
    ```sql
    CREATE TABLE IF NOT EXISTS sub_mods (
        id              INTEGER PRIMARY KEY AUTOINCREMENT,
        mod_id          INTEGER NOT NULL REFERENCES mods(id) ON DELETE CASCADE,
        name            TEXT NOT NULL,
        folder_name     TEXT NOT NULL,
        enabled         INTEGER NOT NULL DEFAULT 0,
        user_enabled    INTEGER NOT NULL DEFAULT 0
    );
    CREATE INDEX IF NOT EXISTS idx_sub_mods_mod_id ON sub_mods(mod_id);
    ```

    `user_enabled` tracks user intent separately from effective state. When parent mod is disabled, `enabled` goes to 0 but `user_enabled` stays unchanged. When parent re-enables, sub-mods where `user_enabled=1` get `enabled=1` restored.

    **Migration version 6** -- `add_sub_mod_id_to_file_entries`:
    ```sql
    ALTER TABLE file_entries ADD COLUMN sub_mod_id INTEGER REFERENCES sub_mods(id) ON DELETE CASCADE;
    CREATE INDEX IF NOT EXISTS idx_file_entries_sub_mod ON file_entries(sub_mod_id);
    ```

    `sub_mod_id` is nullable -- NULL means the file belongs to the parent mod, non-NULL means it belongs to that sub-mod.

    Update the "Next available version" comment at the top to version 7.

    Update the existing `migration_count_is_four` test to `migration_count_is_six` and assert count is 6. The `migration_versions_unique` test should pass unchanged.
