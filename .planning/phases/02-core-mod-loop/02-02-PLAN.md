    Create `src-tauri/src/services/toggle.rs` with these public async functions:

    ```rust
    use std::path::Path;
    use sqlx::SqlitePool;
    use tauri::AppHandle;
    use crate::db::queries;
    use crate::error::AppError;
    use crate::services::{file_ops, journal::FilePair};

    /// Toggle a mod on or off. Wraps all file moves in a journal for crash safety.
    /// When disabling: also disables all sub-mods (preserving user_enabled).
    /// When enabling: restores sub-mods where user_enabled=true.
    pub async fn toggle_mod(
        app: &AppHandle,
        pool: &SqlitePool,
        mod_id: i64,
        enable: bool,
    ) -> Result<(), AppError>

    /// Toggle a single sub-mod on or off. Parent must be enabled.
    /// Sets both enabled and user_enabled to the target state.
    pub async fn toggle_sub_mod(
        app: &AppHandle,
        pool: &SqlitePool,
        sub_mod_id: i64,
        enable: bool,
    ) -> Result<(), AppError>

    /// Permanently delete a mod: remove files from disk (wherever they are), delete DB records.
    pub async fn delete_mod(
        app: &AppHandle,
        pool: &SqlitePool,
        mod_id: i64,
    ) -> Result<(), AppError>
    ```

    **toggle_mod implementation:**
    1. Get mod record and game record from DB
    2. Get file entries for mod (where sub_mod_id IS NULL -- parent files only)
    3. Build FilePair list: if enable, src=staged_path/relative_path, dst=mod_dir/relative_path; if disable, reverse
    4. Call begin_toggle to create journal entry
    5. For each pair: call file_ops::move_file, then mark_file_done
    6. Call complete_journal
    7. Call update_mod_enabled
    8. If disabling: for each sub-mod, set enabled=0 (keep user_enabled as-is), and move sub-mod files to staging too
    9. If enabling: for each sub-mod where user_enabled=1, set enabled=1, move their files to game dir

    **toggle_sub_mod implementation:**
    1. Get sub-mod record, verify parent mod is enabled (error if not)
    2. Get file entries for this sub_mod_id
    3. Build FilePair list using parent mod's staged_path and game's mod_dir
    4. Journal-wrap the file moves
    5. Update sub_mod enabled and user_enabled to the target state

    **delete_mod implementation:**
    1. Get mod record and game record
    2. Get all file entries (including sub-mod files)
    3. For each file: try removing from game dir path, try removing from staging path (whichever exists)
    4. Clean up empty directories under staging
    5. Delete mod from DB (cascade handles file_entries and sub_mods)

    Also add helper functions:
    ```rust
    /// Build file pairs for a list of file entries, given source and destination base dirs.
    pub fn build_file_pairs(entries: &[FileEntry], src_base: &Path, dst_base: &Path) -> Vec<FilePair>
    ```

    Update `src-tauri/src/services/mod.rs` to add `pub mod toggle;`

    For tests: test `build_file_pairs` as a pure function (no async, no DB needed). The async toggle/delete functions will be tested via integration tests in the commands layer.
