  <name>Task 1: Create ModList, ModCard, and SubModOptions components</name>
  <files>src/components/ModList.tsx, src/components/ModCard.tsx, src/components/SubModOptions.tsx, src/App.tsx</files>
  <action>
    **ModList.tsx:**
    - Container component that takes `gameId` prop
    - Uses `useMods(gameId)` to fetch mod list
    - If no mods: render `<EmptyModView />` (existing component)
    - If mods exist: render header bar with game name and "Import" button (top-right per user decision)
    - Render mod cards sorted: enabled first, then alphabetical (backend already sorts this way)
    - Import button click: uses `@tauri-apps/plugin-dialog` `open()` to pick a .zip file, then opens ImportDialog
    - Track which mod card is expanded via local state (only one expanded at a time)

    **ModCard.tsx:**
    - Card-based layout using existing `Card` component from `src/components/ui/card.tsx`
    - Shows: mod name, toggle switch (use a styled checkbox or Radix Switch if available, otherwise a button), file count, enabled/disabled pill
    - Toggle switch calls `useToggleMod().mutate({ modId, enable: !mod.enabled })`
    - BEFORE toggling ON: call `useCheckConflicts` -- if conflicts exist, show ConflictDialog instead of toggling directly
    - Clicking the card body (not the toggle) expands/collapses inline details
    - Expanded view shows:
      - File manifest list from `useModFiles(modId)` -- show relative_path for each file
      - Sub-mod options section (if sub-mods exist) via `<SubModOptions />`
      - Delete button (red, with confirmation) -- calls `useDeleteMod().mutate(modId)`
    - Conflict badge: if `useCheckConflicts` returns items, show a warning icon/badge on the card
    - Clicking conflict badge opens ConflictDialog showing conflict details
    - Grey out / disable the toggle switch while a mutation is in-flight (prevent rapid double-clicks per Pitfall 6)

    **SubModOptions.tsx:**
    - Takes `modId` and `parentEnabled` props
    - Uses `useSubMods(modId)` to fetch sub-mod list
    - Renders a list of sub-mod options under an "Options" heading
    - Each option has: name, individual toggle switch
    - Toggle calls `useToggleSubMod().mutate({ subModId, enable: !subMod.enabled })`
    - When `parentEnabled` is false, all toggles are disabled (greyed out) -- parent must be on first
    - Multiple options can be enabled simultaneously (no radio behavior per user decision)

    **App.tsx update:**
    - Replace the current content area (which shows EmptyModView) with `<ModList gameId={activeGameId} />`
    - ModList internally handles the empty state

    Use Tailwind classes throughout. Dark theme (dark-only app). Use lucide-react icons: `ToggleLeft`/`ToggleRight` for toggle state, `AlertTriangle` for conflict badge, `FileText` for file count, `Trash2` for delete, `Upload` for import button.

    IMPORTANT per user decisions:
    - Card-based layout (NOT table)
    - Sort: enabled mods first, then alphabetical
    - Click card body to expand (NOT the toggle)
    - Delete button inside expanded card details
    - Import button top-right in header area
  </action>

    IMPORTANT per user decisions:
