  <name>Task 1: Generate Ed25519 signing keypair and install dependencies</name>
  <files>~/.tauri/modtoggler.key, ~/.tauri/modtoggler.key.pub, src-tauri/Cargo.toml, package.json</files>
  <action>
1. Create the ~/.tauri/ directory if it does not exist.

2. Generate the Ed25519 signing keypair using the Tauri CLI. Run:
   ```
   npx tauri signer generate -w ~/.tauri/modtoggler.key
   ```
   When prompted for password, provide empty string (press Enter twice) -- passwordless per user decision.

   IMPORTANT: Capture the public key output from stdout. It will also be written to ~/.tauri/modtoggler.key.pub. Save this public key string -- it is needed in Task 2 for tauri.conf.json.

3. Verify both key files exist: ~/.tauri/modtoggler.key and ~/.tauri/modtoggler.key.pub

4. Add Rust dependencies to src-tauri/Cargo.toml under [dependencies]:
   ```toml
   tauri-plugin-updater = "2"
   tauri-plugin-process = "2"
   ```

5. Install JS guest bindings in the project root:
   ```
   npm install @tauri-apps/plugin-updater @tauri-apps/plugin-process
   ```

IMPORTANT: Do NOT commit the private key (~/.tauri/modtoggler.key) to git. It lives outside the repo.
  </action>

  </verify>
  <done>Ed25519 keypair files exist at ~/.tauri/modtoggler.key and .key.pub. Cargo.toml has tauri-plugin-updater and tauri-plugin-process. package.json has @tauri-apps/plugin-updater and @tauri-apps/plugin-process.</done>
</task>

<task type="auto">
  <name>Task 2: Configure updater plugin, register plugins, and enable signed builds</name>
  <files>src-tauri/src/lib.rs, src-tauri/tauri.conf.json, src-tauri/capabilities/default.json</files>
  <action>
1. Read the public key from ~/.tauri/modtoggler.key.pub (single line of base64 text).

2. Update src-tauri/tauri.conf.json:
   - Add `"createUpdaterArtifacts": true` inside the existing `"bundle"` object (alongside "active", "targets", "icon").
   - Add a `"plugins"` top-level key with the updater config block:
     ```json
     "plugins": {
       "updater": {
         "pubkey": "PASTE_ACTUAL_PUBLIC_KEY_HERE",
         "endpoints": [
           "https://github.com/SwirlClinic/ModToggler/releases/latest/download/latest.json"
         ],
         "windows": {
           "installMode": "passive"
         }
       }
     }
     ```
   - Use the actual public key content from step 1, NOT a placeholder.
   - Keep `"targets": "all"` unchanged per user decision.
   - Do NOT set bundle.windows.nsis.installMode to "perMachine" (breaks auto-update per Tauri issue #7184). The default "currentUser" is correct.

3. Update src-tauri/src/lib.rs:
   - Add two plugin registrations in the builder chain, AFTER the existing `.plugin(tauri_plugin_fs::init())` line and BEFORE `.manage(state::AppState::default())`:
     ```rust
     .plugin(tauri_plugin_updater::Builder::new().build())
     .plugin(tauri_plugin_process::init())
     ```
   - Note: updater uses Builder pattern (not init()), process uses init(). This matches the research and official docs.

4. Update src-tauri/capabilities/default.json:
   - Add `"updater:default"` and `"process:default"` to the permissions array:
     ```json
     "permissions": [
       "core:default",
       "sql:default",
       "dialog:default",
       "fs:default",
       "updater:default",
       "process:default"
     ]
     ```

5. Run `cargo check --manifest-path src-tauri/Cargo.toml` to verify everything compiles.
  </action>

Project compiles without errors.</done>
