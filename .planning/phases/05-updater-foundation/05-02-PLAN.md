<task type="auto">
  <name>Task 1: Run signed build and verify .sig artifact generation</name>
  <files></files>
  <action>
1. Set the signing environment variables for the build:
   ```bash
   export TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.tauri/modtoggler.key)"
   export TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
   ```

   IMPORTANT: Both env vars must be set. Even with a passwordless key, TAURI_SIGNING_PRIVATE_KEY_PASSWORD must be set to empty string or the build may hang waiting for input.

2. Run the full build:
   ```bash
   cargo tauri build
   ```

   This will take ~2 minutes. The build should complete without errors.

3. After build completes, verify .sig files were generated. Look in:
   - `src-tauri/target/release/bundle/nsis/` for `.nsis.zip` and `.nsis.zip.sig` files
   - The `.sig` file proves the signing pipeline is working

4. Verify the app binary exists and can launch (optional quick check -- the build success is the primary signal).

If the build fails:
- Check for "signature verification failed" errors (public key mismatch)
- Check for hanging at signing step (missing TAURI_SIGNING_PRIVATE_KEY_PASSWORD)
- Check for version mismatch warnings between tauri.conf.json, Cargo.toml, and package.json (all should be 0.1.0)
  </action>
  <verify>
    <automated>ls src-tauri/target/release/bundle/nsis/*.sig 2>/dev/null | head -3</automated>
  </verify>
  <done>.sig signature files exist alongside NSIS installer in the build output directory. Build completed without errors.</done>
</task>
