<task type="auto">
  <name>Task 1: Create release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/` directory and the `release.yml` workflow file.

The workflow must include these steps in order:

1. **Trigger:** `on: push: tags: ['v*']` -- no branch restriction.

2. **Concurrency:** `group: release-${{ github.ref }}` with `cancel-in-progress: true` to prevent duplicate releases if tags are pushed rapidly.

3. **Permissions:** `contents: write` on the job so GITHUB_TOKEN can create releases.

4. **Checkout:** `actions/checkout@v4`.

5. **Extract version from tag:** Shell step using `bash` that strips the `v` prefix from `GITHUB_REF_NAME` and writes `VERSION` to `$GITHUB_ENV`.

6. **Patch version files:** A `shell: bash` step that:
   - Uses `node -e` to patch `package.json` (set `.version` to `$VERSION`) and `src-tauri/tauri.conf.json` (set `.version` to `$VERSION`). Node.js is always available on the runner; do NOT use `jq` (not available on windows-latest).
   - Uses `sed -i` to patch `src-tauri/Cargo.toml` -- replace only the FIRST `version = "..."` line (the `[package]` version). Use the `0,/pattern/{s/pattern/replacement/}` first-match-only sed syntax to avoid accidentally changing dependency version lines.

7. **Commit version bump back:** Configure git as `github-actions[bot]`, stage the 3 patched files, and push to the default branch (`${{ github.event.repository.default_branch }}`). Guard with `git diff --staged --quiet || ...` to skip commit if versions already match. This push is a plain commit to a branch, NOT a tag push, so it will NOT re-trigger this workflow.

8. **Setup Node.js:** `actions/setup-node@v4` with `node-version: 'lts/*'` and `cache: 'npm'`.

9. **Install Rust:** `dtolnay/rust-toolchain@stable`.

10. **Rust cache:** `swatinem/rust-cache@v2` with `workspaces: './src-tauri -> target'`.

11. **Install frontend dependencies:** `npm ci`.

12. **Build and release:** `tauri-apps/tauri-action@v0` (NOT @v1 -- v1 tag does not exist, per research verification). Environment variables:
    - `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`
    - `TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}`
    - `TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}`

    Inputs:
    - `tagName: v__VERSION__` (reads version from tauri.conf.json)
    - `releaseName: 'ModToggler v__VERSION__'`
    - `releaseDraft: false` (updater requires published releases)
    - `prerelease: false`
    - `updaterJsonPreferNsis: true` (NSIS installer for passive updates)
    - `generateReleaseNotes: true` (auto-generated from commits/PRs)

Use `@v0` for tauri-action per research finding that @v1 does not exist. Note in a YAML comment: "Using @v0 per research -- @v1 tag does not exist despite some docs referencing it".

Reference the complete workflow example in 06-RESEARCH.md "Code Examples > Complete Workflow (Recommended)" section for the full structure.
  </action>
