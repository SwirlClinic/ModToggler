  <action>
    Create `src/components/UpdateBanner.tsx` following the IntegrityAlert pattern:

    **Structure:** A blue-themed alert banner that renders conditionally based on updateStore status.

    **When status is 'available':**
    - Blue banner (border-blue-500/40, bg-blue-500/10) with Download icon from lucide-react
    - Text: "Version {update.version} is available" with "Install now" button (underlined text link)
    - Collapsible release notes: ChevronDown/ChevronUp toggle that reveals update.body in a `<pre className="whitespace-pre-wrap text-xs text-blue-300/80 mt-2 max-h-48 overflow-auto">` block
    - Dismiss X button (calls store.dismiss())
    - Note below Install button: "The app will briefly close to install." (per Windows NSIS Pitfall 2)

    **When status is 'downloading':**
    - Banner text: "Downloading update..."
    - Progress bar: Tailwind div with bg-blue-500 width set to `(downloaded/contentLength)*100%`
    - If contentLength is 0: use CSS animation (animate-pulse on the bar at 100% width) for indeterminate state
    - No dismiss button during download

    **When status is 'installing':**
    - Banner text: "Installing update... The app will restart."
    - No buttons (operation in progress)

    **handleInstall function:**
    - Gets store via useUpdateStore.getState()
    - Calls store.setDownloading()
    - Calls update.downloadAndInstall(onEvent) with event handler:
      - 'Started': store.setContentLength(event.data.contentLength ?? 0)
      - 'Progress': store.addProgress(event.data.chunkLength)
      - 'Finished': store.setInstalling()
    - After downloadAndInstall resolves: call relaunch() from @tauri-apps/plugin-process
    - Wrap in try/catch: on error, call store.setError(String(err))
